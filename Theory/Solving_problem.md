# Solving a Problem

A simple approach to tackle any programming problem in this course.

## Steps

1. **Understand the problem** — read the description carefully and note inputs, outputs, and constraints
2. **Ask questions** — clarify anything unclear before you start coding
3. **Gather requirements** — list what the program must do
4. **Design** — break the problem into smaller parts and solve each part one at a time
5. **Write and test** — implement the program and run it with sample input
6. **Test all scenarios** — include edge cases and invalid input where relevant
7. **Optimize** — improve readability or performance only after it works correctly

## Example

**Problem:** Print the multiplication table of 5.

1. Calculate `5 * 5`
2. Print the result as `5 * 5 = 25`
3. Repeat these steps 10 times to print the full table of 5 (from `5 * 1 = 5` to `5 * 10 = 50`)
